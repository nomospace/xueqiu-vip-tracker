# 雪球大V动态监听系统 PRD

> 版本：V1.0  
> 日期：2026-03-16  
> 作者：产品部

---

## 一、项目背景

个人投资者需及时跟踪雪球（Xueqiu）平台大V的观点更新与持仓变动，但手动刷新效率低且易错过关键信息。本系统旨在通过自动化工具，实现大V动态、自选持仓的集中监控与实时提醒，辅助投资决策。

---

## 二、目标用户

- **核心用户**：关注雪球大V的个人投资者
- **使用场景**：
  - 日常查看大V观点
  - 跟踪持仓调仓动作
  - 接收异动提醒

---

## 三、功能需求

### 3.1 大V管理模块

| 功能点 | 描述 |
|--------|------|
| 添加大V | 支持输入雪球用户ID/昵称，自动抓取并保存大V基础信息 |
| 大V列表 | 展示已监听大V，支持按粉丝数/添加时间排序 |
| 删除大V | 移除监听，同时删除相关历史数据（可选保留） |
| 大V详情 | 展示历史动态、当前持仓及变动记录 |

### 3.2 动态监听模块

| 功能点 | 描述 |
|--------|------|
| 内容抓取 | 定时抓取大V发帖、转发、评论（含文字/图片/链接） |
| 去重逻辑 | 通过消息ID/时间戳判断，避免重复存储 |
| 动态时间线 | 按时间倒序展示所有大V动态，支持筛选 |

### 3.3 持仓监控模块

| 功能点 | 描述 |
|--------|------|
| 持仓抓取 | 定时获取大V公开组合的持仓股票、仓位比例 |
| 变动检测 | 对比新旧数据，识别加仓/减仓/新增/清仓动作 |
| 变动历史 | 展示大V调仓时间线，支持按股票/时间筛选 |

### 3.4 通知推送模块

| 功能点 | 描述 |
|--------|------|
| 实时提醒 | WebSocket 推送新动态/持仓变动至前端 |
| 阈值设置 | 支持自定义"仓位变动 > X%"才触发提醒 |
| 渠道扩展 | 预留邮件/钉钉/企业微信推送接口 |

### 3.5 数据可视化模块

| 功能点 | 描述 |
|--------|------|
| 持仓图表 | 饼图展示仓位分布，柱状图展示历史变化 |
| 动态统计 | 发帖频率、互动数据趋势 |

---

## 四、非功能需求

### 4.1 性能要求

- 爬虫频率：默认每15分钟，支持自定义（最低5分钟）
- 页面加载：动态列表首屏 < 2秒
- 推送延迟：新动态生成后 < 5秒推送到前端

### 4.2 稳定性要求

- 爬虫容错：请求失败自动重试3次
- 数据持久化：数据库每日自动备份
- 服务可用性：> 99%

### 4.3 安全性要求

- 访问控制：简单登录验证
- 数据加密：敏感配置存储在阿里云 KMS
- 爬虫合规：控制请求频率，避免对服务器造成压力

## 五、技术架构

### 5.1 技术选型

| 层级 | 技术栈 | 说明 |
|------|--------|------|
| 前端 | Angular 17+ + Tailwind CSS + ECharts | 组件化开发，图表可视化 |
| 后端 | Python 3.11 + FastAPI | 异步高性能框架 |
| 数据库 | PostgreSQL 15 + Redis 7 | 结构化数据 + 缓存 |
| 任务调度 | Celery + Celery Beat | 定时爬虫任务 |
| 部署 | 阿里云 ECS + Docker + Nginx | 容器化部署 |

### 5.2 系统架构

```
[Angular前端] ←→ [Nginx] ←→ [FastAPI后端]
                               ↓
                    [Celery任务队列] ←→ [Redis]
                               ↓
                        [PostgreSQL]
                               ↓
                    [爬虫模块 (Playwright)]
```

## 六、数据库设计

### 6.1 大V表 (vip_users)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL | 主键 |
| xueqiu_id | VARCHAR(50) | 雪球用户ID |
| nickname | VARCHAR(100) | 昵称 |
| avatar | VARCHAR(500) | 头像URL |
| followers | INTEGER | 粉丝数 |
| description | TEXT | 简介 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

### 6.2 动态表 (vip_posts)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL | 主键 |
| vip_id | INTEGER | 大V ID (FK) |
| post_id | VARCHAR(50) | 雪球帖子ID |
| type | VARCHAR(20) | 类型：post/repost/comment |
| content | TEXT | 内容 |
| images | JSONB | 图片列表 |
| created_at | TIMESTAMP | 发布时间 |
| crawled_at | TIMESTAMP | 抓取时间 |

### 6.3 持仓表 (vip_holdings)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL | 主键 |
| vip_id | INTEGER | 大V ID (FK) |
| stock_code | VARCHAR(20) | 股票代码 |
| stock_name | VARCHAR(50) | 股票名称 |
| position | DECIMAL(5,2) | 仓位比例(%) |
| updated_at | TIMESTAMP | 更新时间 |

### 6.4 持仓变动表 (holding_changes)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL | 主键 |
| vip_id | INTEGER | 大V ID (FK) |
| stock_code | VARCHAR(20) | 股票代码 |
| change_type | VARCHAR(20) | 类型：add/remove/increase/decrease |
| old_position | DECIMAL(5,2) | 原仓位 |
| new_position | DECIMAL(5,2) | 新仓位 |
| change_percent | DECIMAL(5,2) | 变动幅度 |
| detected_at | TIMESTAMP | 检测时间 |

## 七、API 设计

### 7.1 大V管理

```
GET    /api/vip              # 获取大V列表
POST   /api/vip              # 添加大V
GET    /api/vip/{id}         # 获取大V详情
DELETE /api/vip/{id}         # 删除大V
```

### 7.2 动态监听

```
GET    /api/posts            # 获取动态列表
GET    /api/posts/{id}       # 获取动态详情
WebSocket /ws                # 实时推送
```

### 7.3 持仓监控

```
GET    /api/holdings/{vip_id}     # 获取大V持仓
GET    /api/changes               # 获取变动历史
```
